T016 — Manual smoke notes for `spec-kitty doctor restart-daemon` (WP03)

Scope of this file
==================

These steps describe the operator-facing smoke an installable dev CLI
build must produce after WP03 lands. Implementation of WP03 itself does
not require spawning a real daemon — the regression suite (T014) covers
the exit-code matrix via monkeypatched primitives. T016 is the
post-merge / pre-release confirmation a reviewer or operator runs by
hand.

Procedure
=========

Run in a clean tmp project (no prior `sync now` / daemon state):

1. `spec-kitty sync now`
   - Expect: exit 0, a sync daemon is running, `~/.spec-kitty/sync-daemon`
     and the owner record at `<sync_root>/daemon/owner.json` are present.
   - Record: PID via `spec-kitty sync status` (the "Daemon PID" row).

2. `spec-kitty doctor restart-daemon`
   - Expect: exit 0, stdout reports "Sync daemon restarted (previous_pid=…,
     new_pid=…)" with new_pid != previous_pid.

3. `spec-kitty sync status --check`
   - Expect: exit 0, daemon is healthy, `package_version` and
     `executable_path` rows match the foreground CLI (no D-3 mismatch).

4. Trigger a boundary mismatch (e.g. briefly run a stale `spec-kitty`
   binary in a sibling venv so the daemon binds to that path, then run
   the current binary in the original shell):
   - Expect: `spec-kitty sync status --check` refuses with the remediation
     line "Run `spec-kitty doctor restart-daemon` to restart the daemon
     at the foreground version/source, then verify with `spec-kitty sync
     status --check`."
   - Run the command from the message; expect exit 0 and the boundary
     to clear.

5. From a clean tmp project where `sync now` was never invoked:
   - `spec-kitty doctor restart-daemon`
   - Expect: exit 1, stdout mentions "spec-kitty sync now".

Acceptance
==========

All five steps produce the expected outcomes. NFR-002 budget (≤10s
end-to-end on a dev machine) is comfortably under because each step is
either an existing primitive call (stop_sync_daemon ~1–5s, daemon
respawn ~1–2s) or a deterministic file-system check.

Reviewer note: T016 is *operator-facing*, not blocking on implementation.
The regression suite in `tests/specify_cli/cli/commands/test_doctor_restart_daemon.py`
mechanically covers each exit-code arm.
