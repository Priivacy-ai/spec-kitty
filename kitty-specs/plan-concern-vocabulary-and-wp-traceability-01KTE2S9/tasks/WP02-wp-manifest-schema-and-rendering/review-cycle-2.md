# WP02 Review Cycle 1 — Changes Requested

## Blocker: tasks-packages/prompt.md violates FR-009

**Issue**: The implementation added `plan_concern_refs` to the WP prompt file frontmatter template in `src/doctrine/missions/mission-steps/software-dev/tasks-packages/prompt.md`. This directly contradicts FR-009.

**FR-009 states (verbatim)**:
> The prompt must NOT include `plan_concern_refs` in WP prompt file frontmatter templates — `WPMetadata` (which parses WP prompt frontmatter) is configured `extra="forbid"` and will raise `ValidationError` on `finalize-tasks --validate-only` if unknown fields appear in frontmatter. `plan_concern_refs` is a `wps.yaml`-only field.

**What was changed (incorrectly)**:

In `src/doctrine/missions/mission-steps/software-dev/tasks-packages/prompt.md`, the implementation added `plan_concern_refs` to the WP prompt frontmatter template block (lines ~110, ~205) and added the instruction "Copy `plan_concern_refs` from the `wps.yaml` entry verbatim into the WP prompt frontmatter."

**Why this breaks things**:

`WPMetadata` in `src/specify_cli/status/wp_metadata.py` is defined with `model_config = ConfigDict(extra="forbid")`. It does not include `plan_concern_refs` as a declared field. When `finalize-tasks --validate-only` reads WP prompt files generated by agents following the updated prompt, it will attempt to parse their frontmatter through `WPMetadata`, and any WP file with `plan_concern_refs` in frontmatter will raise a `ValidationError`. This is an explicit data integrity violation described in the spec.

**How to fix**:

1. Remove `plan_concern_refs` from the WP prompt frontmatter template block in `tasks-packages/prompt.md`.
2. Remove the instruction "Copy `plan_concern_refs` from the `wps.yaml` entry verbatim into the WP prompt frontmatter."
3. Replace with an instruction that makes the distinction clear, e.g.:
   > `plan_concern_refs` lives in `wps.yaml` only — do NOT copy it into WP prompt frontmatter. `WPMetadata` (which parses WP prompt files) uses `extra="forbid"` and will reject it.

The `wps.yaml` example block that shows `plan_concern_refs` at lines ~186-190 is correct and should be kept — that is showing the `wps.yaml` entry, not a WP prompt frontmatter.
