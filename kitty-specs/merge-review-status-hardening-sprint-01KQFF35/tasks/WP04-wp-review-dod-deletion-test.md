## Objective

Add a "deletion test" checklist item to the WP review source template. This item requires reviewers to verify that each error-path test is reachable — i.e., that the test would fail if the implementation fix were deleted.

The change is purely to the source template file. The generated agent copies (`.claude/commands/`, `.amazonq/prompts/`, etc.) must NOT be touched — they are regenerated by `spec-kitty upgrade`.

## Context

**Why this matters**: During mission `auth-tranche-2-5-cli-contract-consumption-01KQEJZK`, a P0 bug reached production because a test validated the structure of an exception handler (it checked that a `try/except` existed) but did not verify the handler was actually reachable. The test passed even without the implementation fix. The deletion test — "if I delete the fix, does this test fail?" — is a 30-second manual check that catches this entire class of bug.

**What the deletion test is**:
1. Write down the implementation change (the fix).
2. Temporarily delete it.
3. Run the associated test.
4. If the test still passes, the error path is untested — the test validates structure, not behaviour.
5. Restore the fix.
