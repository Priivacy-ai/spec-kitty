## Subtask T015 — Add `_check_mission_branch()` to `merge.py`

**Purpose**: Standalone, testable function that checks whether the expected mission branch exists.

**Add** to `src/specify_cli/cli/commands/merge.py` (near other preflight helpers):

```python
def _check_mission_branch(
    mission_slug: str,
    repo_root: Path,
) -> tuple[bool, dict[str, str] | None]:
    """
    Check whether the expected mission branch exists locally.

    Returns:
        (True, None) if the branch exists.
        (False, blocker_payload) if the branch is missing.

    blocker_payload keys: ready, blocker, expected_branch, remediation.
    Does NOT create the branch (dry-run must remain read-only).
    """
    expected_branch = f"kitty/mission-{mission_slug}"
    if _has_branch_ref(expected_branch, repo_root):
        return True, None

    # Get the current HEAD of the merge target branch for the remediation command.
    import subprocess
    try:
        result = subprocess.run(
            ["git", "rev-parse", "HEAD"],
            cwd=repo_root,
            capture_output=True,
            text=True,
            check=True,
        )
        base_sha = result.stdout.strip()[:12]  # short SHA for readability
    except subprocess.CalledProcessError:
        base_sha = "<base-commit>"

    blocker_payload = {
        "ready": False,
        "blocker": "missing_mission_branch",
        "expected_branch": expected_branch,
        "remediation": f"git branch {expected_branch} {base_sha}",
    }
    return False, blocker_payload
```

**Validation**:
- `_check_mission_branch("my-feature-01KQXXX", Path("."))` returns `(True, None)` when the branch exists.
- Returns `(False, {...})` with the four required keys when the branch is absent.
- `mypy --strict` passes. Note: `dict[str, str]` — all values are strings (including `"ready": "false"` is wrong; `ready` must be a `bool`, so the dict type needs adjustment: use `dict[str, str | bool]`).
