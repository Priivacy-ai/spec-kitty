- **D9 ‚Äî WP06 spurious-authoring-staleness: option (b) DECIDED** (data-model.md Landmine 2 extension, alphonso MINOR-3). Freshness stays grounded on the whole-file ``charter.yaml`` content-identity hash (`charter.bundle.compute_bundle_content_hash`), NOT a narrower `catalog`‚Üî`activation` parity check. Consequence: an authoring-only edit (e.g. hand-editing `governance`/`directives`/`activation` without touching the derived `catalog`) trips `synthesized_drg` to `stale` even though nothing *derived* drifted ‚Äî accepted as expected, not a defect. Rationale: (1) always healable by the very next `spec-kitty charter synthesize`/`resynthesize` (which recomputes the hash from current content and re-stamps the manifest) ‚Äî a transient, self-clearing state, never a permanent-stale dead-end like the retired #2758 missing-file trap; (2) option (a) (grounding freshness on a NEW `catalog`‚Üî`activation` parity check) would require net-new logic in `consistency_check.py`/`bundle.py`, outside WP06's owned surface (`computer.py` + 8 test files) and duplicating the #2759 parity-guard shape this same WP retires as moot ‚Äî reintroducing a second signal the mission explicitly collapses into one. Tested: `tests/specify_cli/charter_freshness/test_computer.py::test_synthesized_drg_remediation_clears_genuine_content_change`, `tests/specify_cli/charter_runtime/test_freshness_activation_visibility.py::test_charter_yaml_edit_without_reconcile_is_stale` + `test_reconcile_after_edit_returns_to_fresh`.
- **D10 ‚Äî #2759 retirement confirmed structurally moot, not just removed.** `_activation_parity_drift_reason`/`_PARITY_DRIFT_REMEDIATION` deleted from `computer.py` (T026). The config<->references/graph divergence they guarded against cannot exist once activation relocates into `charter.yaml` (D1/D5) ‚Äî the same file the content hash already covers ‚Äî so the unified content-hash comparison in `_synthesized_drg_graph_state` subsumes the guarantee the parity read used to provide (any edit to `charter.yaml`, activation-shaped or not, flips the hash). `#2758`'s `first_missing_bundle_file` stays out of scope (owned by WP01/`bundle.py`, WP03's caller in `_synthesis.py`).
- **D11 ‚Äî `_compute_charter_source` (Landmine 2) never returns `"stale"`.** Retired outright, not re-homed: `charter.yaml` cannot carry a self-referential hash of itself (chicken-egg), so there is no meaningful upstream artifact left to compare `charter.yaml` against at the `charter_source` layer. States: `missing` (absent) / `invalid` (present, unparseable) / `fresh` (parses). The content-drift question moved entirely to `synthesized_drg` (D9/D10). `synced_bundle` mirrors `charter_source` (`_BUNDLE_FILES` narrowed to `("charter.yaml",)`, contracts/manifest-v2.md M1/M3) ‚Äî `missing` when absent, `stale` when present-but-`charter_source`-`invalid`, `fresh` otherwise.
- **D12 ‚Äî Fallout beyond WP06's 8 owned test files, fixed as necessary consequence, not scope creep.** `tests/specify_cli/charter_preflight/_fixtures.py` (added `seed_charter_yaml()`; wired into `make_fresh_repo`), `tests/specify_cli/charter_preflight/test_runner.py` (`test_built_in_only_passes`, re-pinned `test_stale_charter_blocks`‚Üí`test_invalid_charter_yaml_blocks`), and `tests/specify_cli/charter_runtime/test_freshness_residue.py` (`_seed_residue_repo`, re-pinned `test_genuine_invalid_still_blocks_preflight`) were broken either by WP01's `BUNDLE_CONTENT_HASH_FILES` narrowing landing before this WP, or directly by this WP's `_compute_charter_source` rework ‚Äî none of the 9 WPs claim these files. Left unfixed they would have regressed 4 then-passing tests (confirmed via a before/after `pytest` run: 3 pre-existing reds from WP01 ‚Üí 0; then my own change would have pushed 25 passing ‚Üí far fewer without this fix). Fixed with the smallest mechanical delta (a new fixture helper + swapping which state a scenario constructs), not a redesign of those files' intent.
