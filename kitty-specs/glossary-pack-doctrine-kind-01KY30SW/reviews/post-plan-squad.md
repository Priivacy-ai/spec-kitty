# Post-Plan Adversarial Squad — Glossary Pack Doctrine Kind (Mission A)

Date: 2026-07-21. Four profile-loaded lenses reviewed the plan/spec/design against the **real
code** (not just the artifacts): architecture (architect-alphonso), doctrine/charter integration
(doctrine-daphne), boundary/duplication (paula-patterns), anti-laziness (reviewer-renata).
Findings cross-corroborated with file:line evidence and were independently re-verified by the
orchestrator against source before folding.

## Verdict entering the fold

**Not executable as written.** The enum/repository/URN spine and the "copy `directive` wiring"
thesis for the *data-model + repository* are sound and confirmed. But the **resolution/default-on
path was architecturally wrong**, and several activation surfaces and the migration schema were
under-specified. All findings are concrete, correctable surface errors — no architectural
dead-ends.

## Confirmed findings + resolutions (folded into spec/plan/data-model/contracts)

### Blockers (resolution path)

- **B1 — graph fragment mislocated.** `load_built_in_graph` globs `src/doctrine/*.graph.yaml`
  **non-recursively at the package root** (`drg/loader.py:100,110-131`); all 10 existing fragments
  live there. A fragment under `glossary_packs/built-in/` is never discovered → pack loads but
  never resolves (the exact NFR-003 trap). **Resolution:** ship
  `src/doctrine/glossary_pack.graph.yaml` at the doctrine root; it is **generated by the
  extractor**, not hand-authored — the WP regenerates and commits it. (IC-04)
- **B2 — source-node emission missing.** `_KIND_MAP` maps ref-types to `NodeKind` for **edge
  targets only**; a kind's own nodes are emitted by explicit per-kind blocks in
  `extract_artifact_edges` (`extractor.py`). That function is at the `# noqa: C901` ceiling.
  **Resolution:** add a new emission block (extract a helper to stay ≤15 complexity, NFR-004) that
  globs `glossary_packs/built-in/*.glossary-pack.yaml` and emits `glossary_pack:<id>` nodes. (IC-04)

### Default-on mechanism (was implicit; now named)

- **F2/HIGH — default-on rides `_BUILTIN_ARTIFACT_KINDS`, which no drift-guard protects.** The
  real guard (`tests/doctrine/test_org_pack_augmentation.py:411-431`) binds only `_ALLOWED_KINDS ↔
  _ORG_DRG_CANONICAL_KINDS ∪ mission-type`. `_BUILTIN_ARTIFACT_KINDS` is the set that makes a kind
  active when `activated_kinds` config is absent — omitting it silently filters every glossary node
  out of resolution with a **green** suite. **Resolution:** the default-on mechanism is stated
  explicitly = (three-state `None` default) + (`_BUILTIN_ARTIFACT_KINDS` membership) + (root graph
  fragment); the drift-guard is extended to a genuine **three-way** equality, plus a positive
  "`glossary_packs` ∈ default activated_kinds" assertion. This is the load-bearing DIRECTIVE_043
  gate for the kind. (IC-03)

- **M3 — `charter/consistency_check.py` `_CLI_KIND_TO_DRG_SINGULAR`** (NOT derived) — add

### Path / naming corrections

- **M1/F4 — `org_pack_loader` is at `src/doctrine/drg/org_pack_loader.py`** (not `src/charter/`);
  `_ORG_DRG_CANONICAL_KINDS = frozenset(_ORG_DRG_KIND_ALIASES.keys())` — edit the **alias dict**,
  not a frozenset.
- **M4 — naming reconciled to `glossary_packs`.** Every existing kind has plural == dir ==
  accessor (`service._built_in_dir` = `<root>/<plural>/built-in`, dirs `directives`, `tactics`,
  `agent_profiles`, …). **Decision:** the package dir and accessor are **`glossary_packs`**
  (`src/doctrine/glossary_packs/`, `DoctrineService.glossary_packs`, activation key
  `activated_glossary_packs`), NOT `glossaries`. This preserves the canonical plural==dir==accessor
  invariant.
- **M5/F3 — `built-in/` (hyphen), not `builtin/`.** `_built_in_dir` hardcodes `"built-in"`.
- **m2 — the three lists hold plural STRINGS** (`"directives"`, …) — add the string
  `"glossary_packs"`, not the enum member.
- **m1 — anchor fix:** `from_operator_token` is on the `ArtifactKind` enum
  (`artifact_kinds.py:128`), `YAML_KEY_MAP` in `pack_manager.py`; `kind_vocabulary.py` has neither.
  FR-010 normalisation is genuinely **free** (enum-derived) — credit; only the anchor was wrong.

### Migration fidelity (was a vacuous guard)

- **Anti-laziness CRITICAL — the seed carries fields the schema dropped.** Verified: 104 terms;
  `confidence` is a **float** (0.6/0.75/0.9/0.95/1.0), NOT enum/str; the seed also carries
  `see_also` (1), `introduced_in_mission` (2), and populated `synonyms_to_avoid` (3) — 6
  extra-field occurrences the schema omitted while NFR-002 only checked
  surface/definition/confidence/status, so the migration would drop fields and the test would still
  pass. **Resolution:** `GlossaryTerm` carries **all** seed fields (`confidence: float`, plus
  `see_also`, `introduced_in_mission`, `synonyms_to_avoid`, `aliases`, `banned_synonyms`); NFR-002
  becomes a **standing pack⟺seed parity invariant over the full key-set** (live until Mission C
  deletes the seed), not a one-shot migration snapshot; US3 round-trips the 3 real
  `synonyms_to_avoid` terms, not just an authored fixture.
