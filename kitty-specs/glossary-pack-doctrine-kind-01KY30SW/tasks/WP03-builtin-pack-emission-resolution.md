# WP03 — Built-in pack + extractor emission + root graph fragment + parity + regression

## ⚡ Do This First: Load Agent Profile

Run `/ad-hoc-profile-load python-pedro` and adopt it fully. **This is the load-bearing WP** — the
squad's two blockers live here. The resolution mechanics are ratified in `contracts/pack-schema.md §2`
and `reviews/post-plan-squad.md` (B1/B2). Do not improvise the graph mechanism; follow it exactly.

## Objective

Ship the built-in `spec-kitty-core` glossary pack (all 104 terms, all fields) and make it **resolve as
a loaded DRG node** — the exact silent-invisibility trap the mission exists to kill. That requires
(a) a new **source-node emission block** in the extractor, (b) the **generated** graph fragment shipped
at the **doctrine package root**, and (c) a **non-vacuous** resolution guard with a negative-control
arm. The migrated pack stays in **standing full-key parity** with the seed, and the runtime + casing
gate stay green.

## Context — the two blockers (READ CAREFULLY)

- **B1 — fragment location + generation.** `load_built_in_graph()` globs `src/doctrine/*.graph.yaml`
  at the **package root, non-recursively** (`src/doctrine/drg/loader.py:100,110-131`); all 10 existing
  fragments live there. The fragment is **generated by the extractor** (`_write_graph_yaml`), NOT
  hand-authored. Ship `src/doctrine/glossary_pack.graph.yaml` at the root — a nested per-pack file is
  invisible.
- **B2 — source-node emission.** `_KIND_MAP` maps ref-types to `NodeKind` for **edge targets only**.
  A kind's own nodes are emitted by explicit per-kind blocks in `extract_artifact_edges`
  (directives ~332, tactics ~397, …) in `src/doctrine/drg/migration/extractor.py`. Add a **new block**
  globbing `glossary_packs/built-in/*.glossary-pack.yaml` and emitting `glossary_pack:<id>` nodes.
  The function carries `# noqa: C901` (at the complexity ceiling) — **extract a helper** so you do not
  raise its complexity (NFR-004).
- **Seed facts** (verified): 104 terms; `confidence` float; extra fields `see_also`,
  `introduced_in_mission`, `synonyms_to_avoid`. The seed is **READ, never modified** (C-003).

## Subtasks

### T012 — Non-vacuous resolves-as-loaded-node guard (RED-FIRST) · NFR-003

- File: `tests/doctrine/glossary_packs/test_builtin_pack_resolution.py`.
- **Positive arm**: after loading, the built-in `spec-kitty-core` pack resolves as a `glossary_pack`
  node in the compiled DRG/charter references (not merely present on disk).
- **Negative-control arm (the non-vacuity proof)**: with the graph fragment (or the extractor emission)
  absent/mutated, resolution goes **RED**. Implement this by pointing the loader at a temp doctrine
  root without the fragment, or monkeypatching the emission off, and asserting the node does NOT
  resolve. A guard that only asserts "node present" is vacuous and will be rejected.
- RED first (nothing emits the node yet).

### T013 — Extractor emission block + helper · FR-011

- Edit `src/doctrine/drg/migration/extractor.py`: add a new per-kind block in `extract_artifact_edges`
  that globs `glossary_packs/built-in/*.glossary-pack.yaml` and emits a `glossary_pack:<id>` source
  node per pack. **Extract a helper** (e.g. `_emit_glossary_pack_nodes(...)`) to keep the enclosing
  function ≤ 15 complexity. Mirror the directive block's shape.
- Add the `glossary_packs` entry to `_PATH_KIND_PATTERNS`/`_KIND_MAP` only if edges reference packs by
  path — for Mission A, source-node emission is the requirement; note if you touch these.

### T014 — Author + migrate the 104-term built-in pack · FR-006, SC-002

- File: `src/doctrine/glossary_packs/built-in/spec-kitty-core.glossary-pack.yaml`.
- Migrate ALL 104 terms from `.kittify/glossaries/spec_kitty_core.yaml`, carrying **every field**
  (`surface`, `definition`, `confidence` as the float from the seed, `status`, and `see_also`/
  `introduced_in_mission`/`synonyms_to_avoid` where present). `aliases`/`banned_synonyms` may be
  absent/`null` (Mission B populates). Prefer a deterministic transform (a small script or a careful
  hand-migration) — but the pack file is the committed artefact.
- Do **not** modify the seed file.

### T015 — Regenerate + commit the root graph fragment · FR-011, NFR-003

- Run the extractor to regenerate the graph, producing `src/doctrine/glossary_pack.graph.yaml` at the
  doctrine root (alongside `directive.graph.yaml`). Commit the **generated** file. Do not hand-edit it.

### T016 — Standing pack⟺seed full-key parity + synonyms round-trip · NFR-002

- File: `tests/architectural/test_glossary_pack_parity.py`.
- **Standing** (not one-shot) parity: the built-in pack has all 104 terms and stays green until
  Mission C deletes the seed — a live drift guard, so any later edit to one home without the other fails.
- **Squad F2 — the assertion MUST be seed-driven, not pack-driven.** Iterate **every seed term** and,
  for that term, **every key present in the seed** (including the sparse optionals — `see_also` on 1
  term, `introduced_in_mission` on 2, `synonyms_to_avoid` on 3 of the 104) and assert the pack term
  carries the **identical** value. A pack-side comparison (only checking fields present on the pack,
  treating `None == absent`) would let a dropped sparse field pass green — that is the exact vacuity
  trap; do NOT write it that way. `confidence` compares as a float.
- Explicitly assert the 1 `see_also`, 2 `introduced_in_mission`, and 3 `synonyms_to_avoid` seed
  occurrences round-trip into the pack.

### T017 — SC-004 regression guard · SC-004

- File: `tests/architectural/test_glossary_pack_no_regression.py`.
- Assert the pre-existing terminology gates stay green *with the pack shipped*: import/run
  `tests/architectural/test_glossary_canonical_terms.py` behaviour and `test_no_legacy_terminology.py`
  are unaffected (the seed + casing gate are untouched — C-003). Keep it lightweight: assert the seed
  loader still resolves the seed and the casing baseline is unchanged.
- **Squad F4 — hard C-003 seed-integrity guard**: T014's "do not modify the seed" is instruction-only.
  Add a **file-integrity** assertion that pins `.kittify/glossaries/spec_kitty_core.yaml` content
  (e.g. assert a known term count / a content digest is unchanged, or that the seed path is absent
  from the mission diff). This prevents a lossy migration from silently editing the seed to match the
  pack and thereby faking T016 parity green.
