# WP01 — Dep bump + vendored events refresh (spec-kitty-events 4.0.0)

## Objective

Move the `spec-kitty` CLI onto `spec-kitty-events 4.0.0`, which introduces the
interview-flavored DecisionPoint payloads (`DecisionPointOpenedInterviewPayload`,
`DecisionPointResolvedInterviewPayload`, `OriginSurface`, `OriginFlow`,
`TerminalOutcome`) required by every downstream work package. This WP updates the
`pyproject.toml` pin, replaces the vendored `src/specify_cli/spec_kitty_events/`
tree wholesale from the 4.0.0 source, and audits existing DecisionPoint emitters
for the mandatory `origin_surface="adr"` addition.

## Context

- **spec.md FR-016** requires the CLI to move to `spec-kitty-events==4.0.0` and
  refresh the vendored copy.
- **research.md R-6** records the decision rationale: the 4.0.0 event types do not
  exist in 3.3.0; shipping against 3.3.0 would be knowingly mismatched.
- **data-model.md §9** specifies the exact version pin and copy path.
- The vendored path convention is `src/specify_cli/spec_kitty_events/` mirroring the
  package layout from `../spec-kitty-events/src/spec_kitty_events/` (sibling repo).
- Any existing Python code in `src/` that already emits an ADR-style DecisionPoint
  payload MUST add `origin_surface="adr"` to satisfy the 4.0.0 contract.

## Branch strategy

- Planning/base branch: `main`
- Final merge target: `main`
- Execution worktree: per-lane, allocated by finalize-tasks.

### T003 — Audit existing DecisionPoint emitters; add origin_surface="adr"

**Purpose.** Ensure no existing code in this repo emits a DecisionPoint payload without
the `origin_surface` field that 4.0.0 mandates, which would cause a validation failure
at runtime.

**Steps.**
1. Run the audit grep from the repo root:
   ```
   grep -rn \
     "DecisionPointOpenedPayload\|DecisionPointResolvedPayload\|DecisionPointDiscussingPayload\|DecisionPointOverriddenPayload" \
     src/ | grep -v spec_kitty_events/
   ```
2. For each match found:
   a. Open the file.
   b. Find the payload construction call.
   c. Add `origin_surface="adr"` (or `origin_surface=OriginSurface.ADR` if using the
      enum) to the keyword arguments.
   d. Import `OriginSurface` from `specify_cli.spec_kitty_events.decisionpoint` if not
      already imported.
3. If the grep produces zero matches (the likely outcome for V1), document the result
   in a short comment block in the WP verification notes (no file changes needed).
4. Run `ruff check src/` and `mypy` on any files changed in step 2.

**Files.**

  If files are found, add their paths to this WP's `owned_files` list.

## Risks

- **Sibling repo not present on disk.** The cp path `../spec-kitty-events/` assumes the
  sibling repo is checked out. If absent, fall back to extracting from the installed
  wheel. Document the path used.
- **3.3.0 → 4.0.0 breaking changes.** The existing test suite exercises the vendored
  copy; a breaking change in a module other than `decisionpoint` could break tests.
  Investigate any failures before assuming they are pre-existing.
- **Import aliasing.** If the vendored tree is imported via `specify_cli.spec_kitty_events`
  rather than the installed `spec_kitty_events`, ensure both paths point to 4.0.0 to
  avoid version skew.
