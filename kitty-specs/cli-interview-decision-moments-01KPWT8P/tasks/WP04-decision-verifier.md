# WP04 — Decision verifier

## Objective

Ship `verify.py`, which cross-references deferred decisions in `index.json` against
inline `[NEEDS CLARIFICATION: ...] <!-- decision_id: <id> -->` sentinel markers in
`spec.md` and `plan.md`. The verifier returns a structured `VerifyResponse` and exits
non-zero on any drift finding, providing the gate that ensures LLM-authored markers stay
consistent with the Decision Moment ledger. This WP is check-only (no auto-fix, no doc
patching — spec.md C-010, C-006).

## Context

- **spec.md FR-015** defines the three finding types: `DEFERRED_WITHOUT_MARKER`,
  `MARKER_WITHOUT_DECISION`, `STALE_MARKER`.
- **spec.md scenario 3 and 7** demonstrate the deferred marker workflow and drift
  detection.
- **data-model.md §5** specifies the exact sentinel format and verifier regex.
- **data-model.md §6** specifies the verifier behavior, inputs, and rules.
- **contracts/decision_verify_response.schema.json** defines the `VerifyResponse` JSON
  shape (`status`, `deferred_count`, `marker_count`, `findings` array with `kind`,
  `decision_id_or_ref`, `location`, `detail`).
- The verifier reads `index.json` via `store.load_index()` (WP02). It does NOT call the
  service layer.
- Sentinel regex (from data-model.md §5):
  `\[NEEDS CLARIFICATION: [^\]]*\]\s*<!--\s*decision_id:\s*(\S+?)\s*-->`

## Subtasks

### T008 — Create verify.py (marker ↔ decision cross-check)

**Purpose.** Implement the `verify(mission_dir, mission_slug)` function that loads the
index, scans `spec.md` and `plan.md` for sentinel markers, applies the three-rule
cross-check, and returns a `VerifyResponse`.

**Steps.**
1. Create `src/specify_cli/decisions/verify.py`.
2. Import:
   - `re`, `pathlib.Path`, `typing`
   - `from specify_cli.decisions.store import load_index`
   - `from specify_cli.decisions.models import DecisionStatus`
3. Define the sentinel regex as a module-level constant:
   ```python
   SENTINEL_RE = re.compile(
       r'\[NEEDS CLARIFICATION: [^\]]*\]\s*<!--\s*decision_id:\s*(\S+?)\s*-->'
   )
   ```
4. Define a `VerifyFinding` dataclass (or Pydantic model) with fields:
   `kind: str`, `decision_id_or_ref: str`, `location: Optional[str]`, `detail: Optional[str]`.
5. Define a `VerifyResponse` dataclass with fields:
   `status: str`, `deferred_count: int`, `marker_count: int`, `findings: List[VerifyFinding]`.
   `status` is `"clean"` if `findings == []`, else `"drift"`.
6. Implement `scan_markers(doc_path: Path) -> List[Tuple[str, str]]`:
   Returns a list of `(decision_id, location_str)` tuples where `location_str` is
   `"<filename>:L<line_number>"`. Returns `[]` if the file does not exist.
7. Implement `verify(mission_dir: Path, mission_slug: str) -> VerifyResponse`:
   a. Load `index = load_index(mission_dir)`.
   b. Identify `deferred_entries = [e for e in index.entries if e.status == DecisionStatus.DEFERRED]`.
   c. Build `deferred_ids: set[str] = {e.decision_id for e in deferred_entries}`.
   d. Scan `mission_dir / "spec.md"` and `mission_dir / "plan.md"` for markers using
      `scan_markers`. Collect all `(decision_id, location)` pairs into `all_markers`.
   e. Build `marker_ids: dict[str, str] = {dm_id: loc for dm_id, loc in all_markers}`.
   f. Apply the three rules:
      - **Rule 1 (DEFERRED_WITHOUT_MARKER):** For every `id` in `deferred_ids` with no
        corresponding entry in `marker_ids`:
        Append `VerifyFinding(kind="DEFERRED_WITHOUT_MARKER", decision_id_or_ref=id, location=None, detail="No inline marker found in spec.md or plan.md")`.
      - **Rule 2 (MARKER_WITHOUT_DECISION):** For every `marker_id` in `marker_ids` not
        present in `{e.decision_id for e in index.entries}`:
        Append `VerifyFinding(kind="MARKER_WITHOUT_DECISION", decision_id_or_ref=marker_id, location=marker_ids[marker_id], detail="Marker references unknown decision_id")`.
      - **Rule 3 (STALE_MARKER):** For every `marker_id` in `marker_ids` that IS in the
        index but whose entry status is NOT `DEFERRED`:
        Append `VerifyFinding(kind="STALE_MARKER", decision_id_or_ref=marker_id, location=marker_ids[marker_id], detail=f"Decision is in status '{entry.status.value}', not 'deferred'")`.
   g. Return `VerifyResponse(status="clean" if not findings else "drift", deferred_count=len(deferred_entries), marker_count=len(all_markers), findings=findings)`.
8. Export `verify`, `VerifyResponse`, `VerifyFinding`, `SENTINEL_RE` from
   `src/specify_cli/decisions/__init__.py` (extend the exports added by WP02).

### T019 — Unit tests: test_verify.py (three finding kinds)

**Purpose.** Cover all three `VerifyFinding` kinds and the clean-state path with
isolated, fast unit tests.

**Steps.**
1. Create `tests/specify_cli/decisions/test_verify.py`.
2. Use `tmp_path`. Each test creates a minimal mission directory with:
   - `decisions/index.json` (written via `store.save_index`)
   - `spec.md` and/or `plan.md` with controlled sentinel content
3. Write tests covering:
   a. **Clean case:** one deferred decision with a matching marker in `spec.md` →
      `status="clean"`, `findings=[]`, `deferred_count=1`, `marker_count=1`.
   b. **DEFERRED_WITHOUT_MARKER:** one deferred decision, no sentinel in any doc →
      one finding of kind `DEFERRED_WITHOUT_MARKER`.
   c. **MARKER_WITHOUT_DECISION:** `spec.md` contains a sentinel with an unknown
      `decision_id` not in the index → finding `MARKER_WITHOUT_DECISION`.
   d. **STALE_MARKER:** `spec.md` contains a sentinel referencing a decision that is
      now `RESOLVED` (no longer deferred) → finding `STALE_MARKER`.
   e. **Mixed:** two deferred decisions, one has a marker, one does not; plus one stale
      marker → exactly two findings, correct kinds.
   f. **plan.md scan:** marker lives in `plan.md` (not `spec.md`) → found correctly.
   g. **Missing files:** neither `spec.md` nor `plan.md` exist; one deferred decision →
      `DEFERRED_WITHOUT_MARKER` finding (verify doesn't crash on missing doc files).
   h. **Location format:** finding `location` string is `"spec.md:L<N>"` where N is the
      correct line number.

## Risks

- **Regex edge cases.** The sentinel regex assumes the `decision_id` token has no
  spaces. Test with a ULID containing only Crockford characters to confirm no false
  negatives. Also test that a bare `[NEEDS CLARIFICATION: ...]` without the anchor
  comment is NOT matched (the anchor is mandatory per spec.md FR-014).
- **Large doc files.** `scan_markers` reads the entire file into memory. For
  performance with spec.md files up to a few hundred KB this is fine; document that
  assumption.
- **WP02 dependency.** `verify.py` calls `store.load_index` from WP02. If WP02 ships a
  different `load_index` signature, update the import accordingly.

## Reviewer guidance

- Confirm `scan_markers` returns `location` in `"<filename>:L<N>"` format, not just
  the file path (the CLI layer uses this for human-readable output).
- Verify that `STALE_MARKER` fires for decisions in RESOLVED, DEFERRED→RESOLVED, AND
  CANCELED states — any non-DEFERRED status that still has a lingering marker.
- Confirm `MARKER_WITHOUT_DECISION` fires for a marker with a `decision_id` that is
  absent from `index.entries` entirely (not just absent from deferred entries).
