# WP02 — Decisions core — models + store

## Objective

Create the foundation runtime modules for the Decision Moment ledger: `models.py`
(Pydantic models and enums that define the on-disk and in-memory data shapes) and
`store.py` (atomic I/O for `index.json` and `DM-<decision_id>.md` artifacts). All
subsequent WPs depend on these shapes; correctness and atomicity here are the
highest-value properties to lock down first.

## Context

- **spec.md FR-006, FR-018, FR-019** define the on-disk artifact formats.
- **spec.md NFR-002** requires atomic writes (write-to-tmp-then-rename).
- **data-model.md §1** specifies the exact Pydantic model structure.
- **data-model.md §2** specifies `index.json` format, `DM-<id>.md` markdown layout,
  sorting rules (created_at ASC then decision_id ASC), and write pattern.
- **contracts/index_entry.schema.json** is the wire-level contract for index entries.
- Pattern: mirror `src/specify_cli/status/` package layout (models / store / emit /
  service split). Use `python-ulid>=1.1.0` for ULID generation.
- Deterministic JSON: `json.dumps(..., sort_keys=True, indent=2)` + trailing newline.

### T004 — Create models.py (enums + IndexEntry + DecisionIndex)

**Purpose.** Define the runtime data model: all enums, the frozen `IndexEntry` Pydantic
model, and the `DecisionIndex` container that serializes to `index.json`.

**Steps.**
1. Create `src/specify_cli/decisions/models.py`. Follow the exact structure from
   **data-model.md §1**.
2. Define these enums (all `str, Enum`):
   - `OriginFlow`: `CHARTER="charter"`, `SPECIFY="specify"`, `PLAN="plan"`.
   - `DecisionStatus`: `OPEN="open"`, `RESOLVED="resolved"`, `DEFERRED="deferred"`,
     `CANCELED="canceled"`.
   - `DecisionErrorCode`: `MISSING_STEP_OR_SLOT`, `ALREADY_CLOSED`, `TERMINAL_CONFLICT`,
     `NOT_FOUND`, `MISSION_NOT_FOUND`, `VERIFY_DRIFT` (string values per data-model.md §1.1).
3. Define `IndexEntry(BaseModel)` with `model_config = ConfigDict(frozen=True, extra="forbid")`:
   - Fields: `decision_id: str`, `origin_flow: OriginFlow`, `step_id: Optional[str] = None`,
     `slot_key: Optional[str] = None`, `input_key: str`, `question: str`,
     `options: Tuple[str, ...] = ()`, `status: DecisionStatus`,
     `final_answer: Optional[str] = None`, `rationale: Optional[str] = None`,
     `other_answer: bool = False`, `created_at: datetime`, `resolved_at: Optional[datetime] = None`,
     `resolved_by: Optional[str] = None`, `mission_id: str`, `mission_slug: str`.
   - `@model_validator(mode="after")` that raises `ValueError` if both `step_id` and
     `slot_key` are `None`.
4. Define `DecisionIndex(BaseModel)` with `model_config = ConfigDict(frozen=True, extra="forbid")`:
   - Fields: `version: Literal[1] = 1`, `mission_id: str`,
     `entries: Tuple[IndexEntry, ...] = ()`.
5. Define `DecisionOpenResponse` and `DecisionTerminalResponse` dataclasses or Pydantic
   models matching `contracts/decision_open_response.schema.json` and
   `contracts/decision_terminal_response.schema.json`. Fields:
   - `DecisionOpenResponse`: `decision_id`, `idempotent: bool`, `mission_id`,
     `artifact_path`, `event_lamport: Optional[int]`.
   - `DecisionTerminalResponse`: `decision_id`, `status: DecisionStatus`,
     `terminal_outcome: str`, `idempotent: bool`, `event_lamport: Optional[int]`.
6. Create `src/specify_cli/decisions/__init__.py` that exports the public symbols
   (`IndexEntry`, `DecisionIndex`, `OriginFlow`, `DecisionStatus`, `DecisionErrorCode`,
   `DecisionOpenResponse`, `DecisionTerminalResponse`).

### T005 — Create store.py (atomic index.json + DM-<id>.md I/O)

**Purpose.** Implement all filesystem I/O for the decisions sub-directory: reading and
writing `index.json` (atomically), creating/updating `DM-<decision_id>.md` artifacts,
and providing the idempotency-key lookup used by the service layer.

**Steps.**
1. Create `src/specify_cli/decisions/store.py`. Import `os`, `json`, `tempfile`,
   `pathlib.Path`, `datetime`, `typing`, and the models from `models.py`.
2. Implement `decisions_dir(mission_dir: Path) -> Path`:
   Returns `mission_dir / "decisions"`. Does NOT create the directory.
3. Implement `load_index(mission_dir: Path) -> DecisionIndex`:
   - Path: `decisions_dir(mission_dir) / "index.json"`.
   - If file does not exist, return `DecisionIndex(mission_id="", entries=())` (caller
     must pass `mission_id` at write time).
   - Parse JSON and return a `DecisionIndex` instance.
4. Implement `save_index(mission_dir: Path, index: DecisionIndex) -> None`:
   - Serialize with `json.dumps(index.model_dump(mode="json"), sort_keys=True, indent=2)`.
   - Append `"\n"` (trailing newline).
   - Write atomically: `tempfile.NamedTemporaryFile(dir=decisions_dir(...), delete=False)`
     → write bytes → `tmp.flush()` → `os.replace(tmp.name, target_path)`.
   - Create `decisions_dir(mission_dir)` with `mkdir(parents=True, exist_ok=True)` before
     the temp file if it does not exist.
5. Implement `find_by_logical_key(index: DecisionIndex, mission_id: str, origin_flow: OriginFlow, step_or_slot: str, input_key: str) -> Optional[IndexEntry]`:
   - Returns the most recent (by `created_at`) entry matching all four fields.
   - `step_or_slot` matches against `step_id` if `step_id` is not None, else against
     `slot_key`.
6. Implement `append_entry(mission_dir: Path, entry: IndexEntry) -> DecisionIndex`:
   - Load the current index.
   - Create a new `DecisionIndex` with `entries` sorted by `(created_at, decision_id)` ASC.
   - Save atomically.
   - Return the updated index.
7. Implement `update_entry(mission_dir: Path, decision_id: str, **updates) -> DecisionIndex`:
   - Load the current index.
   - Find the entry by `decision_id`. Raise `KeyError` if not found.
   - Create a replacement `IndexEntry` using `entry.model_copy(update=updates)`.
   - Rebuild the `DecisionIndex` with the replacement, re-sort, save atomically.
   - Return the updated index.
8. Implement `write_artifact(mission_dir: Path, entry: IndexEntry) -> Path`:
   - Renders `DM-<decision_id>.md` as human-readable markdown per **data-model.md §2.2**.
   - Includes all fields (decision_id, origin_flow, step_id/slot_key, input_key, question,
     options list, status, final_answer if resolved, rationale if deferred/canceled,
     other_answer, created_at, resolved_at, resolved_by) and a change-log footer.
   - Write atomically (same tmp+replace pattern) to `decisions_dir(mission_dir) / f"DM-{entry.decision_id}.md"`.
   - Return the artifact path.

## Risks

- **Pydantic v1 vs v2 in-repo.** Check whether `src/specify_cli/status/models.py` uses
  Pydantic v1 or v2 API and match the same version; do not introduce a second major
  version.
- **Frozen model mutation in update_entry.** Since `IndexEntry` is frozen, `model_copy`
  is the correct mutation path — verify this works with the in-repo Pydantic version.
- **datetime timezone handling.** All `created_at`/`resolved_at` values must be
  timezone-aware UTC (`datetime.now(timezone.utc)`). Naive datetimes break sort
  stability in tests.
