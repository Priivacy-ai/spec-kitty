# WP03 — Decisions service — emit + orchestration

## Objective

Ship the core contract-producer for Decision Moments: `emit.py` wraps the existing
status event emitter to append `DecisionPointOpened(interview)` and
`DecisionPointResolved(interview)` events to `status.events.jsonl`, and `service.py`
is the public orchestration layer providing `open()`, `resolve()`, `defer()`, and
`cancel()` with idempotency semantics. This WP is the layer that all callers (CLI
commands, charter Python integration) depend on.

## Context

- **spec.md FR-003 – FR-011** define the open/resolve/defer/cancel behavior,
  idempotency semantics, and event emission requirements.
- **spec.md FR-007** requires events to carry `origin_surface=planning_interview`,
  `origin_flow`, `question`, `options`, `input_key`, `step_id` (or `slot_key`), and
  actor metadata — all using the 4.0.0 `DecisionPointOpenedInterviewPayload`.
- **research.md R-3** records the idempotency key design:
  `(mission_id, origin_flow, step_id|slot_key, input_key)`.
- **data-model.md §2.3** specifies the event shapes and the rule that `step_id` on
  the wire is populated from `step_id` if set, else from `slot_key`.
- **contracts/decision_open_response.schema.json** and
  **contracts/decision_terminal_response.schema.json** define the return shapes.
- Import event payload types from `specify_cli.spec_kitty_events.decisionpoint`
  (the vendored 4.0.0 copy). NEVER import from the installed `spec_kitty_events`
  package directly — always use the vendored path for consistency.
- Use `python-ulid` for `decision_id` minting in `service.open()`.
- Do NOT emit `DecisionPointWidened` — that is out of scope (spec.md C-002).
- Do NOT make SaaS network calls (spec.md C-003, FR-017).

### T006 — Create emit.py (DecisionPoint event emission)

**Purpose.** Provide two functions — `emit_decision_opened` and
`emit_decision_resolved` — that construct the 4.0.0 interview-variant payloads and
append them to `status.events.jsonl` via the existing status emitter pipeline.

**Steps.**
1. Create `src/specify_cli/decisions/emit.py`.
2. Import:
   - `from specify_cli.spec_kitty_events.decisionpoint import (DecisionPointOpenedInterviewPayload, DecisionPointResolvedInterviewPayload, OriginSurface, OriginFlow as EventOriginFlow, TerminalOutcome)`
   - `from specify_cli.status.store import append_event` (or whichever function the
     existing status package exposes for appending raw event dicts to `status.events.jsonl`).
   - `from specify_cli.decisions.models import IndexEntry, OriginFlow, DecisionStatus`.
3. Implement `emit_decision_opened(mission_dir: Path, entry: IndexEntry, actor: str) -> int`:
   - Construct `DecisionPointOpenedInterviewPayload` with:
     - `decision_point_id=entry.decision_id`
     - `origin_surface=OriginSurface.PLANNING_INTERVIEW`
     - `origin_flow=EventOriginFlow(entry.origin_flow.value)`
     - `step_id=entry.step_id or entry.slot_key`  (wire field is always `step_id`)
     - `input_key=entry.input_key`
     - `question=entry.question`
     - `options=list(entry.options)`
     - `actor=actor`
   - Serialize the payload to a dict and wrap in the existing event envelope (match
     the shape used by `status.emit` — inspect `src/specify_cli/status/emit.py` for the
     envelope structure: `event_id`, `at`, `event_type`, `payload`, etc.).
   - Append to `mission_dir / "status.events.jsonl"` via the status store append function.
   - Return the line count / lamport counter of the appended event (or 0 if the
     existing API does not expose it).
4. Implement `emit_decision_resolved(mission_dir: Path, entry: IndexEntry, terminal_outcome: str, actor: str) -> int`:
   - Construct `DecisionPointResolvedInterviewPayload` with:
     - `decision_point_id=entry.decision_id`
     - `origin_surface=OriginSurface.PLANNING_INTERVIEW`
     - `terminal_outcome=TerminalOutcome(terminal_outcome)`
     - `final_answer=entry.final_answer`
     - `other_answer=entry.other_answer`
     - `rationale=entry.rationale`
     - `actor=actor`
   - Wrap in event envelope and append.
   - Return lamport counter.

### T007 — Create service.py (open/resolve/defer/cancel orchestration)

**Purpose.** The public Python API for Decision Moments. Callers (CLI commands,
charter integration) call `service.open(...)` etc. rather than using `store` or
`emit` directly.

**Steps.**
1. Create `src/specify_cli/decisions/service.py`.
2. Import `ulid` from `python-ulid`, `datetime.timezone`, and the models, store, and
   emit modules from this package.
3. Implement `open(*, mission_dir: Path, mission_id: str, mission_slug: str, origin_flow: OriginFlow, step_id: Optional[str] = None, slot_key: Optional[str] = None, input_key: str, question: str, options: Sequence[str] = (), actor: str = "cli", dry_run: bool = False) -> DecisionOpenResponse`:
   a. Validate: raise `DecisionError(code=MISSING_STEP_OR_SLOT)` if both `step_id` and
      `slot_key` are None.
   b. Load index. Call `find_by_logical_key(index, mission_id, origin_flow, step_id or slot_key, input_key)`.
   c. If match found and `status` is non-terminal (`OPEN`):
      Return `DecisionOpenResponse(decision_id=match.decision_id, idempotent=True, ...)`.
   d. If match found and `status` is terminal:
      Raise `DecisionError(code=ALREADY_CLOSED, details={"decision_id": ..., "status": ...})`.
   e. If no match (or `dry_run`):
      - If `dry_run`: return a mock response with `decision_id="DRY_RUN"` and `idempotent=False`.
      - Mint ULID: `decision_id = str(ulid.new())`.
      - Build `IndexEntry` with `status=OPEN`, `created_at=datetime.now(timezone.utc)`.
      - `store.append_entry(mission_dir, entry)`.
      - `store.write_artifact(mission_dir, entry)`.
      - `lamport = emit.emit_decision_opened(mission_dir, entry, actor)`.
      - Return `DecisionOpenResponse(decision_id=..., idempotent=False, mission_id=..., artifact_path=..., event_lamport=lamport)`.
4. Implement `resolve(*, mission_dir: Path, decision_id: str, final_answer: str, other_answer: bool = False, rationale: Optional[str] = None, resolved_by: Optional[str] = None, actor: str = "cli", dry_run: bool = False) -> DecisionTerminalResponse`:
   a. Load index. Find entry by `decision_id`. Raise `DecisionError(NOT_FOUND)` if absent.
   b. If already terminal and payload identical → idempotent return.
   c. If already terminal and payload differs → raise `DecisionError(TERMINAL_CONFLICT)`.
   d. If `dry_run`: return mock response.
   e. Build updated entry with `status=RESOLVED`, `final_answer`, `other_answer`,
      `rationale`, `resolved_by`, `resolved_at=datetime.now(timezone.utc)`.
   f. `store.update_entry(mission_dir, decision_id, ...)`.
   g. `store.write_artifact(mission_dir, updated_entry)`.
   h. `lamport = emit.emit_decision_resolved(mission_dir, updated_entry, "resolved", actor)`.
   i. Return `DecisionTerminalResponse(status="resolved", terminal_outcome="resolved", ...)`.
5. Implement `defer(*, mission_dir, decision_id, rationale, resolved_by=None, actor="cli", dry_run=False) -> DecisionTerminalResponse`:
   - Same pattern as `resolve` but `terminal_outcome="deferred"`, no `final_answer`.
   - Raise `TERMINAL_CONFLICT` if already terminal with different outcome.
6. Implement `cancel(*, mission_dir, decision_id, rationale, resolved_by=None, actor="cli", dry_run=False) -> DecisionTerminalResponse`:
   - Same pattern, `terminal_outcome="canceled"`.
7. Define a `DecisionError(Exception)` class with `code: DecisionErrorCode` and
   `details: Optional[dict]` fields — used by the CLI layer for structured JSON errors.
8. Export all public symbols via `__init__.py` updates (add `service`, `emit` to
   `src/specify_cli/decisions/__init__.py`).

### T018 — Unit tests: test_service_terminal.py (resolve/defer/cancel + conflict)

**Purpose.** Verify terminal command semantics: idempotency on exact re-call, conflict
rejection on contradictory re-call, and that each terminal outcome updates the index
and artifact correctly.

**Steps.**
1. Create `tests/specify_cli/decisions/test_service_terminal.py`.
2. Use `tmp_path`; stub emit functions.
3. Write tests covering:
   a. `service.resolve(...)` transitions status to `RESOLVED` and writes `final_answer`
      to the index and artifact.
   b. `service.resolve(...)` called twice with identical `final_answer` is idempotent
      (`idempotent=True`, no new event).
   c. `service.resolve(...)` called after `service.defer(...)` raises
      `DecisionError(code=TERMINAL_CONFLICT)`.
   d. `service.resolve(...)` called twice with different `final_answer` raises
      `DecisionError(code=TERMINAL_CONFLICT)`.
   e. `service.defer(...)` transitions status to `DEFERRED`, records rationale.
   f. `service.cancel(...)` transitions status to `CANCELED`, records rationale.
   g. `service.resolve(decision_id="nonexistent", ...)` raises
      `DecisionError(code=NOT_FOUND)`.
   h. `--dry-run` on each terminal command returns mock response with no filesystem
      side effects.
   i. `service.resolve(...)` with `other_answer=True` records `other_answer=True` in
      the index entry (spec.md scenario 5).
