# WP07 — Template updates + CHANGELOG

## Objective

Update the two SOURCE command templates (`specify.md` and `plan.md` under
`src/specify_cli/missions/software-dev/command-templates/`) to instruct the LLM to call
`spec-kitty agent decision open/resolve/defer/cancel/verify` at ask time and resolution
time during interview phases. Add a CHANGELOG entry documenting the V1 Decision Moments
feature, the events dep bump, and the template changes. This WP completes the LLM-driven
specify/plan integration required by spec.md FR-013 and FR-014.

## Context

- **spec.md FR-013** requires SOURCE templates to instruct the LLM to call `decision
  open` before each interview question and a terminal command after each answer.
- **spec.md FR-014** requires templates to instruct the LLM to write
  `[NEEDS CLARIFICATION: <text>] <!-- decision_id: <id> -->` for deferred decisions.
- **CLAUDE.md (critical, read first):** SOURCE templates live at
  `src/specify_cli/missions/*/command-templates/`. Agent copies (`.claude/commands/`,
  `.amazonq/prompts/`, etc.) are GENERATED and must NOT be edited here. Only edit the
  two source files listed in `owned_files`.
- **data-model.md §8** provides the exact instruction block text to add to both templates.
- The instruction block must cover the full Decision Moment protocol:
  call `decision open` before asking → capture `decision_id` → ask question → call
  exactly one terminal command → if deferred, write inline marker → before finishing
  interview phase, run `decision verify`.
- The 4.0.0 event contract terms that must appear in templates:
  `origin_surface` (value `planning_interview`), `OriginFlow` (values `specify` or
  `plan`), `terminal_outcome`, `DecisionPointOpenedInterviewPayload`,
  `DecisionPointResolvedInterviewPayload` — these are for the template's reference
  section; the LLM does not construct payloads directly but must understand the
  downstream contract.

## Subtasks

### T012 — Update specify.md template (Decision Moment protocol section)

**Purpose.** Add an explicit "Decision Moment Protocol" instruction block to the
`specify.md` command template so that the LLM knows to call the decision CLI before
asking each interview question during the Specify phase.

**Steps.**
1. Open `src/specify_cli/missions/software-dev/command-templates/specify.md` and read
   the full file. Identify the Discovery Gate / interview phase section where the LLM
   is instructed to ask clarifying questions.
2. Locate the instruction that tells the LLM to ask questions (typically near "Interview
   Questions" or "Discovery Gate" or "NEEDS CLARIFICATION" guidance).
3. Insert a new section titled `## Decision Moment Protocol` immediately before or
   within that instruction block. The section must contain:

   ```
   ## Decision Moment Protocol

   Before asking **any** interview question during this command, you MUST:

   1. Run `spec-kitty agent decision open` to mint a decision_id:
      ```
      spec-kitty agent decision open \
        --mission <mission-slug> \
        --flow specify \
        --slot-key specify.<phase>.<question-slug> \
        --input-key <snake_case_key> \
        --question "<question text>" \
        [--options '["option1","option2","Other"]']
      ```
      Capture the returned `decision_id` from the JSON output.

   2. Ask the question to the user in chat.

   3. After the user answers, run **exactly one** of:
      - Resolved answer:
        `spec-kitty agent decision resolve <decision_id> --mission <slug> --final-answer "<answer>" [--other-answer]`
      - Deferred / skip:
        `spec-kitty agent decision defer <decision_id> --mission <slug> --rationale "<reason>"`
      - Not applicable / cancel:
        `spec-kitty agent decision cancel <decision_id> --mission <slug> --rationale "<reason>"`

   4. When deferring, write the inline marker into `spec.md` immediately after the
      relevant section:
      ```
      [NEEDS CLARIFICATION: <brief description of what needs answering>] <!-- decision_id: <decision_id> -->
      ```

   5. Before declaring the interview phase complete, run:
      `spec-kitty agent decision verify --mission <slug>`
      Address any findings (DEFERRED_WITHOUT_MARKER, MARKER_WITHOUT_DECISION,
      STALE_MARKER) before proceeding.

   **Important constraints:**
   - `--slot-key` format: `specify.<section>.<question-slug>` (e.g.,
     `specify.auth.strategy`).
   - `--input-key` is the snake_case programmatic key (e.g., `auth_strategy`).
   - The `decision_id` on the wire is a plain ULID (26 chars). The `DM-` prefix
     appears only in artifact filenames, not in CLI arguments.
   - Do NOT emit `DecisionPointWidened` — that is out of scope for V1.
   - SaaS sync is not required; all operations are local-only.
   ```

4. Verify that the section is placed so it is visible to the LLM before it reaches
   the interview question list (not buried at the bottom of the template).
5. If there is an existing `[NEEDS CLARIFICATION]` instruction block in the template,
   update it to include the `<!-- decision_id: <id> -->` anchor and reference the new
   `decision defer` command rather than just writing a bare marker.
