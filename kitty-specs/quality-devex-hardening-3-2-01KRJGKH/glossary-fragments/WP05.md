# WP05 reinforces but does not introduce canonical terms

WP05 authors behavior-driven coverage tests for `charter.py`, `charter_bundle.py`,
`agent/config.py`, `internal_runtime/engine.py`, and `core/file_lock.py`.

The canonical terms introduced in WP02 apply throughout this WP:

- **characterization test** (from WP02): a test that locks in current observable
  behavior before a refactor, serving as a regression guard. WP05 uses the same
  pattern for the five new test targets (see T023–T028).
- **Bucket A / B / C** split (from WP02 research): WP05 applies the A/B/C
  partitioning (orchestration / filesystem-IO / rendering) to each new test
  target, keeping test concerns separated and maintainable.

See `glossary-fragments/WP02.md` for the canonical definition of "characterization test".
