# WP06 Glossary Fragment

Two canonical terms introduced by WP06 for use across the mission and project.

---

## structural debt

**Definition**: A function whose cognitive complexity has accumulated beyond
reviewable bounds AND whose responsibilities are independently dispatching
concerns with no shared state between branches. A structural debt function is a
refactor candidate per the `refactoring-extract-first-order-concept` and
`refactoring-extract-class-by-responsibility-split` tactics. Distinguished from
deliberate linearity by the *absence* of a documented design intent and by the
presence of independent dispatch arms or triplicated patterns.

**Canonical example in this mission**: `doctor.py::mission_state` at CC 57
(three independent dispatch arms — `--audit`, `--fix`, `--teamspace-dry-run` —
sharing no state after option parsing, plus triplicated JSON-vs-pretty emit
logic). Classified as structural debt in `work/findings/refactor-audit.md`
(WP06 resolution: extracted to CC 3 thin orchestrator + per-mode runners).

**Confidence**: 0.9
**Status**: active

---

## deliberate linearity

**Definition**: A function whose body is intentionally kept long-and-linear
(rather than extracted into named helpers) for traceability against a documented
contract, code-review map, or output-section sequence. Deliberate linearity
functions are **not** refactor candidates regardless of Sonar S3776 score.
Identified by an explicit code comment justifying the structure and/or a
published output-section contract that reviewers verify line-by-line.
Refactoring a deliberately linear function without updating its contract is a
regression in traceability.

**Canonical example**: `src/specify_cli/cli/commands/_auth_doctor.py::render_report`
(CC 53; opens with `# The 7-section layout intentionally stays linear so
reviewers can map each block to the contract without bouncing through helper
indirection`). Classified as deliberate linearity in `work/findings/refactor-audit.md`;
left untouched per constraint C-003 (auth-maintainer sign-off required).

**Confidence**: 0.95
**Status**: active
