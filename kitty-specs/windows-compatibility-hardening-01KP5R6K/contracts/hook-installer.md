# Contract: Pre-commit Hook Installer (Windows-safe)

**Spec IDs**: FR-009, FR-010, FR-011, C-003
**Module**: `src/specify_cli/policy/hook_installer.py`

## `install(repo_root: Path, *, interpreter: Path | None = None) -> HookInstallRecord`

```python
def install(repo_root: Path, *, interpreter: Path | None = None) -> HookInstallRecord:
    """Install the Spec Kitty pre-commit hook into `<repo_root>/.git/hooks/pre-commit`.

    The hook pins the absolute Python interpreter at install time and invokes
    `python -m specify_cli.policy.commit_guard_hook` via that interpreter.
    No PATH lookup. No `python3` / `python` / `py` assumption.
    """
```

## Generated hook contents (canonical)

```sh
#!/bin/sh
# Generated by spec-kitty. Do not edit by hand.
# Installed: 2026-04-14T10:35:00Z
# Interpreter: C:\Users\alice\pipx\venvs\spec-kitty-cli\Scripts\python.exe
set -e
exec "<ABSOLUTE_INTERPRETER_PATH>" -m specify_cli.policy.commit_guard_hook "$@"
```

Where `<ABSOLUTE_INTERPRETER_PATH>` is:
- On POSIX: `Path(sys.executable).resolve()` as-is (e.g. `/home/user/.local/pipx/venvs/spec-kitty-cli/bin/python`).
- On Windows: `Path(sys.executable).resolve()` in its native form (e.g. `C:\Users\alice\pipx\venvs\spec-kitty-cli\Scripts\python.exe`). Git for Windows' bundled `sh.exe` will accept either the native form (via its MSYS path translation) or a `/c/Users/...` form; the installer uses the native form and validates via execution test.

## Guarantees

- G-01: The generated hook file contains exactly one quoted, absolute interpreter path.
- G-02: The hook file's first line is exactly `#!/bin/sh`.
- G-03: The hook is mode `0o755` (`stat.S_IRWXU | stat.S_IRGRP | stat.S_IXGRP | stat.S_IROTH | stat.S_IXOTH`).
- G-04: The generated hook executes successfully on:
  - POSIX with `/bin/sh` (existing behavior).
  - Git for Windows default install (MSYS `sh.exe` invoked via git) against a pipx-installed spec-kitty.
- G-05: The generated hook executes successfully when the interpreter path contains a space (e.g. `C:\Program Files\Python311\python.exe`).
- G-06: The hook invokes `specify_cli.policy.commit_guard_hook` and propagates its exit code.
- G-07: Re-installing is idempotent: previous hook is replaced atomically via temp-file-then-rename.
