# WP06 — Pre-commit hook installer hardening

## Branch strategy

- Planning base branch: `main`
- Final merge target: `main`
- Execution worktree: per-lane `.worktrees/windows-compatibility-hardening-01KP5R6K-lane-<id>/`.
- Implement command: `spec-kitty agent action implement WP06 --agent <name>`. No dependencies — start in parallel with WP01.

## Objective

Rewrite the git pre-commit hook generator in `src/specify_cli/policy/hook_installer.py` so the generated hook pins the absolute Python interpreter at install time (captured from `sys.executable`) and does not rely on `python`/`python3`/`py` being on PATH. Validate actual executable behavior — on `windows-latest` (Git for Windows' bundled MSYS `sh.exe`), including interpreter paths that contain spaces and PATH stripped of Python launchers (reproduces issue #105).

## Context

- **Spec IDs covered**: FR-009 (pin absolute interpreter), FR-010 (executable across Windows install topologies), FR-011 (tests validate execution), C-003 (no POSIX/PATH assumptions).
- **Discovery decision**: Q3=A (single `#!/bin/sh` hook with pinned `sys.executable`; no `.cmd` companion).
- **Historical context**: Issue #105 ("Commit operation failed on Windows because `python3` was assumed") was closed narrowly; this WP adds the structural fix + native regression test.
- **Contract**: [`contracts/hook-installer.md`](../contracts/hook-installer.md)

## Detailed subtasks

### T031 — Capture absolute `sys.executable` at install time

**Purpose**: The generated hook must embed a concrete path, not a PATH lookup.

**Steps**:
1. Open `src/specify_cli/policy/hook_installer.py`.
2. Locate the `install(...)` function (or equivalent entrypoint that writes the hook file).
3. At the top of the function:
   ```python
   import sys
   from pathlib import Path

   interpreter = Path(sys.executable).resolve(strict=False)
   if not interpreter.is_file():
       raise RuntimeError(
           f"Cannot install pre-commit hook: sys.executable={sys.executable!r} "
           f"resolves to {interpreter}, which does not exist."
       )
   ```
4. Pass `interpreter` to the rendering function in T032.

**Validation**:
- T035 reads the installed hook and confirms the interpreter path appears literally.

### T032 — Rewrite hook template

**Purpose**: Single `#!/bin/sh` file; one quoted `exec` line; no PATH lookups.

**Steps**:
1. Replace the existing hook body template with:
   ```python
   HOOK_TEMPLATE = """#!/bin/sh
   # Generated by spec-kitty. Do not edit by hand.
   # Installed: {installed_at}
   # Interpreter: {interpreter}
   set -e
   exec "{interpreter}" -m specify_cli.policy.commit_guard_hook "$@"
   """
   ```
2. Render with:
   ```python
   from datetime import datetime, timezone
   rendered = HOOK_TEMPLATE.format(
       installed_at=datetime.now(timezone.utc).isoformat(timespec="seconds"),
       interpreter=str(interpreter),
   )
   ```
3. The `exec` line MUST:
   - Use double-quotes around `{interpreter}` so paths with spaces work (`C:\Program Files\Python311\python.exe`).
   - Use `exec` so the hook does not fork an extra shell layer.
   - Use `-m specify_cli.policy.commit_guard_hook` to invoke the module entrypoint.
   - Pass `"$@"` to forward arguments.
4. Do NOT add any fallback logic (C-003, CLAUDE.md rule: no fallbacks). The interpreter path is either correct or the hook fails loudly.

**Validation**:
- T035 tests rendered shape.
- T036/T037/T038 test actual execution.

### T033 — Atomic install via temp-file-then-rename; mode `0o755`

**Purpose**: Re-install must not leave a partial hook file. Hook must be executable on POSIX.

**Steps**:
1. In `install(...)`, write to a temp file first:
   ```python
   import os, tempfile
   hooks_dir = repo_root / ".git" / "hooks"
   hooks_dir.mkdir(parents=True, exist_ok=True)
   fd, tmp_path = tempfile.mkstemp(prefix="pre-commit.", dir=str(hooks_dir))
   try:
       with os.fdopen(fd, "w", encoding="utf-8", newline="\n") as f:
           f.write(rendered)
       os.chmod(tmp_path, 0o755)
       os.replace(tmp_path, str(hooks_dir / "pre-commit"))
   except Exception:
       try:
           os.unlink(tmp_path)
       except OSError:
           pass
       raise
   ```
2. The `newline="\n"` argument is critical — Git for Windows' `sh.exe` does not tolerate CRLF shebang lines.
3. `0o755` is the POSIX executable bit. It's a no-op on NTFS but harmless; do not conditionalize.

**Validation**:
- T035 asserts mode is 0o755.
- Interrupted install leaves no half-written hook (verified via T035's atomic-replace assertion).

   ```python
       mode=0o755,
   )
   ```

### T035 — Rendering tests: shebang, single quoted interpreter line, mode [P]

**Purpose**: Pin the rendered hook shape.

**Steps**:
1. Create `tests/policy/test_hook_installer_rendering.py`:
   ```python
   import os, sys, stat
   from pathlib import Path

   from specify_cli.policy.hook_installer import install


   def test_hook_rendering_shape(tmp_path):
       # Fake repo with .git dir
       repo = tmp_path / "repo"
       (repo / ".git").mkdir(parents=True)

       record = install(repo)
       hook = repo / ".git" / "hooks" / "pre-commit"
       assert hook.is_file()

       content = hook.read_text(encoding="utf-8")
       lines = content.splitlines()
       assert lines[0] == "#!/bin/sh"

       exec_lines = [l for l in lines if l.startswith("exec ")]
       assert len(exec_lines) == 1
       exec_line = exec_lines[0]
       # Quoted absolute interpreter path
       assert f'"{sys.executable}"' in exec_line or f'"{Path(sys.executable).resolve()}"' in exec_line
       assert "-m specify_cli.policy.commit_guard_hook" in exec_line
       assert '"$@"' in exec_line

       # Mode 0o755
       mode = stat.S_IMODE(os.stat(hook).st_mode)
       assert mode == 0o755

       # Record fields
       assert record.shebang == "#!/bin/sh"
       assert record.module == "specify_cli.policy.commit_guard_hook"
       assert record.interpreter == Path(sys.executable).resolve()
       assert record.mode == 0o755
   ```
2. No `windows_ci` marker — reads the file on any platform.

**Validation**:
- Passes on POSIX.

### T036 — Windows execution test: actual hook runs [P]

**Purpose**: Prove the hook is executable on Git for Windows.

**Steps**:
1. Create `tests/policy/test_hook_installer_execution.py`:
   ```python
   import os, subprocess, sys
   from pathlib import Path
   import pytest

   from specify_cli.policy.hook_installer import install


   def _init_git(repo: Path) -> None:
       subprocess.run(["git", "init", "--quiet"], cwd=repo, check=True)
       subprocess.run(["git", "config", "user.email", "test@example.com"], cwd=repo, check=True)
       subprocess.run(["git", "config", "user.name", "Test"], cwd=repo, check=True)


   @pytest.mark.windows_ci
   def test_hook_executes_on_windows(tmp_path):
       repo = tmp_path / "repo"
       repo.mkdir()
       _init_git(repo)
       install(repo)

       (repo / "file.txt").write_text("hello")
       subprocess.run(["git", "add", "file.txt"], cwd=repo, check=True)

       # The commit guard should run and either pass (return 0) or fail with
       # a non-127/non-126 code. 127 = "command not found" (PATH lookup broken).
       # 126 = "command not executable" (mode/shebang broken).
       proc = subprocess.run(
           ["git", "commit", "-m", "test"],
           cwd=repo,
           capture_output=True,
           text=True,
       )
       assert proc.returncode not in (126, 127), (
           f"Hook invocation failed at shell level. rc={proc.returncode}\n"
           f"stdout={proc.stdout}\nstderr={proc.stderr}"
       )
   ```
2. Adjust repo init steps if Spec Kitty's `install(...)` has project-layout prerequisites (e.g., a `.kittify/` directory). Create those in `tmp_path` before calling `install`.

**Validation**:
- Runs on `windows-latest`.

### T037 — Windows execution with path-with-spaces [P]

**Purpose**: The most common real-world Windows failure mode.

**Steps**:
1. In the same `test_hook_installer_execution.py`, add:
   ```python
   @pytest.mark.windows_ci
   def test_hook_executes_with_spaces_in_interpreter_path(tmp_path, monkeypatch):
       # Symlink/copy the current interpreter into a path with a space
       spaced = tmp_path / "My Programs" / "Python"
       spaced.mkdir(parents=True)
       import shutil
       fake_interp = spaced / Path(sys.executable).name
       shutil.copy2(sys.executable, fake_interp)

       monkeypatch.setattr(sys, "executable", str(fake_interp))

       repo = tmp_path / "repo"
       repo.mkdir()
       _init_git(repo)
       install(repo)

       (repo / "file.txt").write_text("hello")
       subprocess.run(["git", "add", "file.txt"], cwd=repo, check=True)
       proc = subprocess.run(
           ["git", "commit", "-m", "test-spaces"],
           cwd=repo,
           capture_output=True,
           text=True,
       )
       assert proc.returncode not in (126, 127), (
           f"Hook invocation failed with interpreter at a path containing a space. "
           f"rc={proc.returncode}\nstdout={proc.stdout}\nstderr={proc.stderr}"
       )
   ```
2. Note: copying the interpreter to a new location may not fully "just work" if the interpreter depends on relative-path resources (stdlib location). Use a symlink on platforms that support them, and fall back to copy otherwise. On Windows runners without Developer Mode, symlinks may not work — copy is the robust option, combined with setting `PYTHONHOME` if necessary. If copy-based simulation proves too fragile, use the native GitHub-hosted Python install path which happens to contain a space on certain runner generations (verify on `windows-latest`).

**Validation**:
- Runs on `windows-latest`.

### T038 — Regression test for issue #105 [P]

**Purpose**: Even with `python`/`python3`/`py` absent from PATH, the hook must execute.

**Steps**:
1. Create `tests/regressions/__init__.py` (empty).
2. Create `tests/regressions/test_issue_105_hook_python_lookup.py`:
   ```python
   import os, subprocess, sys
   from pathlib import Path
   import pytest


   @pytest.mark.windows_ci
   def test_hook_executes_with_python_stripped_from_path(tmp_path, monkeypatch):
       from specify_cli.policy.hook_installer import install

       repo = tmp_path / "repo"
       repo.mkdir()
       subprocess.run(["git", "init", "--quiet"], cwd=repo, check=True)
       subprocess.run(["git", "config", "user.email", "t@e.com"], cwd=repo, check=True)
       subprocess.run(["git", "config", "user.name", "T"], cwd=repo, check=True)
       install(repo)

       # Strip python launchers from PATH
       orig_path = os.environ.get("PATH", "")
       cleaned = os.pathsep.join(
           p for p in orig_path.split(os.pathsep)
           if not any(
               Path(p).joinpath(name + ext).is_file()
               for name in ("python", "python3", "py")
               for ext in ("", ".exe")
           )
       )
       env = {**os.environ, "PATH": cleaned}

       (repo / "file.txt").write_text("regression-105")
       subprocess.run(["git", "add", "file.txt"], cwd=repo, check=True)
       proc = subprocess.run(
           ["git", "commit", "-m", "regression-105"],
           cwd=repo, env=env, capture_output=True, text=True,
       )
       assert proc.returncode not in (126, 127), (
           f"Regression of #105: hook fails when python/python3/py not on PATH. "
           f"rc={proc.returncode}\nstdout={proc.stdout}\nstderr={proc.stderr}"
       )
   ```

**Validation**:
- Runs on `windows-latest`.

## Risks

- **Git-for-Windows MSYS path translation**: The bundled `sh.exe` rewrites drive-letter arguments. Use the native `C:\...\python.exe` form as-is inside double quotes; MSYS handles this correctly in shebang-bang lines and in quoted exec arguments. If T036 fails due to MSYS translation, switch to the `/c/Users/...` POSIX form computed at install time (R-05 backup plan).
- **Developer Mode required for symlinks on Windows**: T037's spaced-path fixture uses copy, not symlink, to avoid this dependency.
- **Copied interpreter may lose stdlib**: If `shutil.copy2(sys.executable, ...)` creates an interpreter that can't import the stdlib (because of relative-path resolution), set `PYTHONHOME` to the original interpreter prefix for the duration of the test.
- **Existing hook_installer callers**: Verify no caller relies on the function returning `None`. The `HookInstallRecord` return is additive.
