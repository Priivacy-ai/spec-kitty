# WP01 — Replace stale allowlist in diagnose with canonical registry union

**Spec anchors**: FR-001..FR-007, NFR-001..NFR-004, C-001..C-007.
**Anchor issue**: `Priivacy-ai/spec-kitty#1222`.

## Scope

Single behavioural change to `src/specify_cli/sync/diagnose.py`:

1. Remove the import of `VALID_EVENT_TYPES` from `emitter`.
2. Add a private import of `_EVENT_TYPE_TO_MODEL` from
   `spec_kitty_events.conformance.validators`, with a comment block
   explaining the precedent (already used at
   `src/specify_cli/status/lifecycle_events.py:210`).
3. Compute a module-level `KNOWN_EVENT_TYPES` frozen-set as
   `frozenset(_EVENT_TYPE_TO_MODEL.keys() | _PAYLOAD_RULES.keys())`.
4. Update `_validate_extended_envelope` to test membership against
   `KNOWN_EVENT_TYPES` and to surface a clearer error message ("not in
   canonical registry or local payload rules").
5. Leave `_validate_payload` and its consumption of `_PAYLOAD_RULES`
   alone.

Add tests in `tests/sync/test_diagnose.py`:

1. `TestCanonicalRegistryRecognition.test_recognises_every_registry_type`
   — iterates over the canonical registry and asserts no recognition
   error.
2. `TestCanonicalRegistryRecognition.test_recognises_cli_internal_types`
   — explicit check for the 7 CLI-internal types (`BuildHeartbeat`,
   `BuildRegistered`, `DependencyResolved`, `ErrorLogged`,
   `HistoryAdded`, `MissionOriginBound`, `WPAssigned`).
3. `TestCanonicalRegistryRecognition.test_rejects_genuinely_unknown_type`
   — assert a sentinel string is rejected with the value mentioned.
4. `TestCanonicalRegistryRecognition.test_drift_detector_picks_up_new_registry_entries`
   — monkeypatch the canonical-registry mapping to add a synthetic
   type, reload the diagnose module to recompute `KNOWN_EVENT_TYPES`,
   assert the synthetic type is recognised. This is the drift detector.

## Implementation steps

1. **Test first** — add the four new tests in
   `tests/sync/test_diagnose.py`. Confirm
   `test_recognises_every_registry_type` fails on at least one
   registry type not currently in `_PAYLOAD_RULES`.
2. **Refactor** — change `diagnose.py` per the design in `plan.md`.
3. **Re-run** — `pytest tests/sync/test_diagnose.py -v`. All tests
   green.
4. **Smoke** — `pytest tests/sync/test_forward_compatibility.py -v`
   and `pytest tests/contract/test_handoff_fixtures.py -v`. All green
   (FR-007).
