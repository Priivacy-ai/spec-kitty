## Summary

Redesign `spec-kitty init` from a per-project installer into a global machine-level bootstrapper. Remove 11 CLI flags, 9 processing stages, dead template infrastructure, and the entirely unused `preferred_implementer`/`preferred_reviewer` preference system from the full codebase. Add a new migration (`m_3_2_2`) that implements the safe per-project command removal with all four safety invariants (the existing `m_3_1_2` cannot be hardened in-place ‚Äî see Critical Findings). Author 7 ADRs documenting all decisions made during the overhaul.

## Technical Context

**Language/Version:** Python 3.11+
**CLI framework:** typer
**Console output:** rich
**YAML parsing:** ruamel.yaml
**Testing:** pytest (90%+ coverage on new/modified code)
**Type checking:** mypy --strict (must pass on all modified files)
**Storage:** filesystem only (`.kittify/config.yaml`, migration files, ADR markdown)
**Target platform:** macOS, Linux, WSL

**Critical finding ‚Äî Migration B cannot be hardened in-place:**
`m_3_1_2_globalize_commands.py` lacks all four safety invariants but **cannot be modified in-place**. The migration runner (runner.py:182‚Äì190) tracks applied migrations by `migration_id`. Projects that already ran `"3.1.2_globalize_commands"` will skip any changes to that file. WP04 must create `m_3_2_2_harden_globalize_commands.py` with a new ID. The existing 3.1.2 file is left as-is.

**Critical finding ‚Äî `agents ‚Üí tools` rename vs load_agent_config() mismatch:**
`m_2_0_1_tool_config_key_rename.py` renames `agents` to `tools` in config.yaml, but `load_agent_config()` only reads from `agents`. On post-2.0.1 projects, agent config silently returns empty defaults. WP01 must patch `load_agent_config()` to try `tools` first, then fall back to `agents`. Migration A (WP03) already handles both keys for the selection cleanup; this fix ensures the underlying read path is consistent.

**Critical finding ‚Äî skills installation loses its home in WP02:**
`install_skills_for_agent()` is called inside the per-agent loop that WP02 removes (init.py:1177). `ensure_runtime()` in bootstrap.py copies missions and scripts but does **not** install agent skills. After WP02, nothing calls `_sync_global_skill()` during init. WP02 must add a global skill installation pass to `ensure_runtime()` or a dedicated call after the global runtime bootstrap, before the per-agent loop is removed.

**Key research finding ‚Äî version marker:**
A file is spec-kitty-generated iff its first line starts with `<!-- spec-kitty-command-version:` (written by `shims/generator.py:101`, read by `m_2_1_4_enforce_command_file_state.py:39`).

**Key research finding ‚Äî migration version ordering:**
The migration registry orders by `packaging.version.Version(target_version)`. WP03 uses `3.2.1`, WP04 uses `3.2.2`. No collision.

### WP04 ‚Äî Migration B: New Safe Local Command Removal (`m_3_2_2`)

**Lane:** B ¬∑ **Position:** 2 of 2 ¬∑ **Depends on:** nothing (sequential for lane order only)

**‚öÝ Do NOT modify `m_3_1_2_globalize_commands.py`.** The migration runner tracks applied migrations by `migration_id`. Projects that already ran `"3.1.2_globalize_commands"` will never re-execute changes to that file. A new migration file with a new ID is required.

**Files:**
- `src/specify_cli/upgrade/migrations/m_3_2_2_safe_globalize_commands.py` (NEW)
- `tests/upgrade/migrations/test_m_3_2_2_safe_globalize_commands.py` (NEW)
- `m_3_1_2_globalize_commands.py` ‚Äî **do not touch**

**New migration spec (`m_3_2_2_safe_globalize_commands.py`):**
```
migration_id = "3.2.2_safe_globalize_commands"
target_version = "3.2.2"
description = "Safe removal of per-project spec-kitty command files with global-presence checks"
```

`detect()`: same as 3.1.2 ‚Äî returns True if any `spec-kitty.*` file exists in any configured agent command dir.

`apply()`: iterate configured agents with four safety invariants per-agent.

**Safety invariants (must all pass per-agent before any deletion):**

1. **Global runtime present:** `Path.home() / ".kittify" / "missions"` exists and contains at least one subdirectory
2. **Global agent commands present:** `Path.home() / agent_root / subdir` exists and contains at least one `spec-kitty.*` file
3. **File is spec-kitty-generated:** the local target file's first line starts with `<!-- spec-kitty-command-version:`

**Invariant failure behavior:** skip entire agent, append `f"Skipped {agent_root}: {reason}"` to `changes`. Do not raise; return `MigrationResult(success=True, changes_made=changes)` ‚Äî the migration is advisory, not blocking.

**Remove:** the best-effort `agent_dir.rmdir()` empty-directory cleanup ‚Äî too aggressive.

**New test cases:**
- Global runtime absent (`~/.kittify/missions/` missing) ‚Üí all agents skipped, warning in changes
- Global commands absent for agent `claude` but present for `codex` ‚Üí claude skipped, codex cleaned
- Local file lacks version header ‚Üí that file skipped, sibling files with header removed
- Dry-run with all invariants met ‚Üí "Would remove:" entries, no filesystem changes
- Mixed project (2 agents safe, 1 not) ‚Üí correct per-agent independent behavior
- Existing passing tests continue to pass

**Acceptance gate:**
- `mypy --strict` on migration file ‚Üí clean
- All new + existing migration tests green
