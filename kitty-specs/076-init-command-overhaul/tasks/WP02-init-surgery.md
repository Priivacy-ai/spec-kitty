## Subtask T018 — Fix Error Handling, Add Global Skills, Fix Charter Dir

**Purpose:** Three targeted fixes to make the resulting init command correct.

### Fix 1: ensure_runtime() error handling (FR-003)

Find the try/except block around the `ensure_runtime()` call (~line 1107):
```python
try:
    ensure_runtime()
except Exception as exc:
    logger.debug(f"ensure_runtime failed: {exc}")
    # continues silently
```
Replace with:
```python
try:
    ensure_runtime()
except Exception as exc:
    _console.print(f"[red]Error:[/red] Failed to bootstrap global runtime: {exc}")
    raise typer.Exit(1)
```

### Fix 2: Global skill installation (FR-007)

The per-agent skill install loop (lines ~1165–1191) is being removed in this WP (via removal of the whole per-agent block). This loop was the only call site for `_sync_global_skill()`.

After the `ensure_runtime()` call succeeds, add a global skill installation step:
```python
# Install skills globally
from specify_cli.skills.installer import install_all_global_skills  # or equivalent entry point
try:
    install_all_global_skills(get_kittify_home() / "agent-skills")
except Exception as exc:
    _console.print(f"[yellow]Warning:[/yellow] Skill installation incomplete: {exc}")
    # Non-fatal: skills can be re-installed on next upgrade
```

**Important:** Before writing this, read `src/specify_cli/skills/installer.py` to find the correct batch install entry point. If `install_all_global_skills()` does not exist, create a thin wrapper that iterates the skill registry and calls `_sync_global_skill()` for each discovered skill. Document the entry point you use in a code comment.

### Fix 3: Remove charter/ directory from `_prepare_project_minimal()`

Find `_prepare_project_minimal()` (lines ~173–190). It creates `.kittify/charter/` directory. Remove this line:
```python

```
Charter is per-project governance; the directory should not be created during global machine init.
