# spk-doctrine-charter

Use this skill for project governance, charter creation, charter sync, or
doctrine context questions.

## Flow

1. Invoke `/spec-kitty.charter` or the relevant charter command.
2. Keep governance decisions explicit and versioned.
3. Load charter context on demand, scoped to the active action.
4. Use glossary and SPDD skills when the charter selects those practices.

## Legacy Alias

For detailed doctrine service access and charter command maps, use
`spec-kitty-charter-doctrine` when available.
