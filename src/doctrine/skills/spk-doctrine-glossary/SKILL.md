# spk-doctrine-glossary

Use this skill when the user asks about canonical terms, glossary updates,
terminology drift, or domain language consistency.

## Flow

1. Locate active glossary context.
2. Classify the change: new term, alias, conflict, drift, or usage correction.
3. Update or apply terminology without rewriting unrelated docs.
4. Feed domain language back into spec, plan, and documentation skills.

## Legacy Alias

For detailed glossary workflows and field guides, use
`spec-kitty-glossary-context` when available.
