# 3.2.x — Strangle the naming / identity / read-path split-brain

**Milestone:** [`3.2.x`](https://github.com/Priivacy-ai/spec-kitty/milestone/4) (open) ·
**Status:** active · **Declared:** 2026-06-16 · **Advances:** epics #1868 (canonical seams), #1619
(runtime/state SSOT), #1878 (coordination strangler).

## Goal

Route every consumer of mission identity, names, and read paths onto a **single source of truth**
and **close the enforcement gaps**, so the recurring *"name-vs-authority"* wrong-compose class — which
has been re-fixed across roughly **9 missions / 38 issue references from Feb to Jun 2026** — **cannot
regrow**. Ship it as a series of emergent patches, **introducing no new shadow paths**.

## Why now (the evidence)

A two-squad investigation (static + forensic "Code as a Crime Scene") established:

- **3.2.0 built the right authority** (`lanes/branch_naming.py` is cold and simple — maxCC 8) but the
  defect class did not die: the **disease lives in the consumers** that re-derive identity/paths
  inline. The naming-class defects (175 fix-commits) concentrate in `implement.py`, `agent/tasks.py`,
  `agent/workflow.py` — not the authority modules.
- The **coord/primary read surface is the empirical #1 crime scene** — `/coord/` appears in ~14× the
  commits of `/mid8/`, the resolver files are ~100% fix-churn, and the class is **accelerating**
  while the mid8 class **plateaued** post-3.2.0 (the ratchet held; the topology surface did not).
- The current literal-ban ratchet keys on the token `mid8` and **misses bare `mission_id[:8]`
  derivations** (~20 sites) — a forensically-proven hole through which the class silently regrows
  (e.g. the NNN-strip #1589 fix that had to be re-applied 14 days later).

Full research: [`docs/engineering_notes/naming-identity-ssot-strangler/`](../engineering_notes/naming-identity-ssot-strangler/)
(`00-OVERVIEW.md` + the static, CaaCS, and context-threading findings).

## Scope (the SSOT surface)

Bring each concern onto its canonical authority and route consumers to **consume / thread** it rather
than re-derive:

- **Identity / naming** — `lanes/branch_naming.py` (established 3.2.0); drain the residual composers
  (#2000, #1899-tail, #1900) and **extend the ratchet to bare `mission_id[:8]`** + `parents[N]`.
- **Project root** — `core/paths.py::locate_project_root` (collapse the remaining hops; #1971).
- **Lanes dir** — extract a pure `resolve_lanes_dir()` seam returning *both* surfaces (#1993).
- **Coord/primary read path** — the consolidated read API + Context value objects
  (`_read_path_resolver` / `surface_resolver` / `coordination/types.py`); the **#1 crime scene**.
- **Ownership** — `ownership/validation.py` existence-check (#1888, verify-and-close).

> **Mechanism under active refinement:** an investigation is testing whether the read-path SSOT is
> already a **Context-value-object + consolidated-API** design (compute-once, thread-through) that is
> simply **under-adopted** (consumers re-derive instead of threading) — in which case the work is
> *adoption/threading*, not new construction. The mission spec will pin the exact mechanism from that
> verdict; the goal above is invariant to it.

## Non-goals (the fence)

- The **#1878 coord/primary WRITE/entry-side strangler** (commit / protected-branch durability) — a
  semantics-sensitive, characterization-first effort tracked to **3.3.x**.
- Renaming the `mid8`/`mission_id`/`mission_slug` domain terms (identity model is fixed by mission 083).
- The `merge.py` size/growth problem and `dashboard/scanner.py` refactor (the ratchet must *reach*
  scanner.py repo-wide, but it is not refactored in-cycle — follow-up).
- Legacy-cutover (#1057–1060), ownership-policy (#1766/#1979), bulk-edit, DRG lints.

## Success criteria (cycle closes when all hold)

- [ ] Every naming/identity/path/coord-read consumer routes through the canonical SSOT (no inline
      `mission_id[:8]`, `kitty/mission-{…}`, `.worktrees/…`, or `parents[N]` re-derivation outside the
      seams).
- [ ] The literal-ban ratchet covers the full **recurrence shape** (bare `mid8[:8]`, `parents[N]`,
      assign-then-join) **repo-wide incl. `dashboard/scanner.py`**, and its allow-list **shrinks**.
- [ ] No new shadow path introduced; byte-identical names preserved (the C-LANES-1 three-surfaces
      and strip-vs-verbatim guardrails held).
- [ ] The wrong-compose defect class produces **zero new regressions** for one full cycle after close.

## Emergent patches

- **3.2.1** — identity-SSOT completion (composes drain + the `[:8]` ratchet gap) + the lanes-dir /
  project-root seams + #1888 verify-close. (First, highest-ROL, lowest-risk.)
- **3.2.2** (emergent) — the coord/primary read-path SSOT slice (the #1 crime scene; characterization-
  guarded) + repo-wide ratchet reach. Exact shape pinned by the context-threading verdict.
