# docs/output/

The `documentation` mission type's generic template reserves this path as a deliverables
directory for missions that generate a fresh documentation set into an isolated output tree.

Mission `docs-ia-onboarding-overhaul-01KY02JB` (gap-filling mode) did not use this pattern — its
deliverables are direct in-place edits, moves, and additions inside the existing `docs/` tree
(navigation restructure, new pages, glossary activation tooling), not a separate generated
output tree. This directory exists only to satisfy the mission type's generic path convention
check; it intentionally has no other content.

Full deliverable record: `kitty-specs/docs-ia-onboarding-overhaul-01KY02JB/docs-audit.md` and
`docs/plans/engineering-notes/terminology-sweep-report.md`.
