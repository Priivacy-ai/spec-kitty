# Planning

Cross-mission delivery-planning notes — how upcoming work is sequenced, prioritised, and gated across missions. These are steering documents for the maintainer, not governance records: once the plan they describe has been executed, the page is left as provenance and any durable decision is recorded in an ADR under [`docs/adr/`](../adr/3.x/README.md).

Distinct from [`docs/plans/`](../plans/index.md) (per-mission and per-refactor implementation plans) — this section holds the *portfolio-level* approach that decides which of those run first.

## Pages

- [3.2.x Delivery Approach — Operator Plan, Dialectically Challenged](3-2-x-approach.md) — the 3.2.x sequencing intent (doctrine-swap epics first, code cleanup next, hold non-critical community PRs), stress-tested by a two-round dialectic squad and synthesised into an actionable, fact-checked plan.
